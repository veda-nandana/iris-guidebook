# Perjalanan Program di IRIS

Halaman ini adalah titik mulai untuk memahami IRIS. Bacalah halaman ini sebelum mengikuti panduan menu: IRIS tidak hanya dipakai untuk mengunggah berkas, melainkan untuk mengawal satu program **Penelitian** atau **Pengabdian kepada Masyarakat (PKM)** dari gagasan sampai luarannya terdokumentasi.

[Kunjungi situs resmi DRPM Primakara University](https://drpm.primakara.ac.id/){target="_blank" rel="noreferrer"} untuk informasi program, kegiatan, dan kontak DRPM.

## Cerita singkat: dari ide menjadi dampak

Bayangkan Ibu/Bapak memiliki ide riset atau kegiatan pengabdian. Ketika periode dibuka, ide itu diajukan sebagai proposal di IRIS. Admin memastikan pengajuan siap diproses, kemudian reviewer menilai kelayakannya. Bila proposal diterima, pengusul menjalankan kegiatan dan melaporkan kemajuan secara bertahap. Reviewer memberi penilaian pada tahap yang menjadi tugasnya, sedangkan admin memantau keseluruhan proses. Perjalanan berakhir saat luaran—seperti diseminasi atau publikasi—telah dicatat.

Dengan urutan ini, setiap orang tahu apa yang sedang terjadi, siapa yang perlu bertindak, dan dokumen apa yang perlu disiapkan.

## Alur end-to-end

| Tahap | Cerita proses | Peran utama | Hasil yang dicapai |
| --- | --- | --- | --- |
| 1. Persiapan periode | Admin menyiapkan periode, pengguna, dan rubrik. Pengusul menyiapkan ide, tim, serta proposal. | Admin, Pengusul | Periode siap dan proposal siap diajukan. |
| 2. Pengajuan proposal | Pengusul mengisi informasi program, menambahkan tim, lalu mengirim proposal. | Pengusul | Proposal tercatat dengan status diproses. |
| 3. Verifikasi & penugasan | Admin memeriksa kelengkapan. Jika lengkap, admin menetapkan reviewer dan jenis review. | Admin | Proposal siap dinilai atau dikembalikan untuk dilengkapi. |
| 4. Review & keputusan | Reviewer membaca proposal, memberi nilai dan catatan. Pengusul melakukan revisi bila diminta; keputusan akhir kemudian ditetapkan. | Reviewer, Pengusul, Admin | Proposal diterima, perlu revisi, atau ditolak. |
| 5. Pelaksanaan kegiatan | Proposal yang diterima menjadi kegiatan aktif. Pengusul menjalankan rencana dan mengunggah laporan kemajuan sesuai jadwal. | Pengusul | Bukti kemajuan tersedia untuk dipantau. |
| 6. Monitoring & evaluasi | Reviewer menilai dokumen tahap pelaksanaan yang telah diserahkan; admin memantau nilai serta kelengkapan. | Reviewer, Admin | Catatan evaluasi dan perkembangan kegiatan tercatat. |
| 7. Penutupan & luaran | Pengusul menyelesaikan laporan akhir, cek plagiarisme, diseminasi, dan publikasi sesuai ketentuan periode. | Pengusul, Reviewer, Admin | Luaran program terdokumentasi dan kegiatan dapat ditutup. |

## Peta status yang mudah dibaca

Pada tahap proposal, status menunjukkan posisi proses, bukan nilai akhir pengusul:

`Diproses` → diperiksa admin → `Lengkap`/ditugaskan ke reviewer → `Diterima`, `Revisi`, atau `Ditolak`.

Jika berstatus **Revisi**, baca catatan yang diberikan, perbaiki proposal, lalu ajukan kembali. Setelah **Diterima**, fokus berpindah dari proposal ke pelaksanaan dan pelaporan tahap demi tahap.

### Arti status secara rinci

| Status | Arti |
| --- | --- |
| `diproses` | Proposal telah dikirim dan sedang diperiksa. |
| `revisi` | Reviewer meminta perbaikan isi proposal. |
| `diajukan_kembali` | Pengusul telah mengirim ulang proposal setelah revisi. |
| `diterima` | Proposal dapat dilanjutkan ke pelaksanaan. |
| `ditolak` | Proposal tidak dilanjutkan pada periode tersebut. |
| `pending` | Dokumen menunggu verifikasi Admin. |
| `revisi_admin` | Admin meminta kelengkapan atau perbaikan dokumen. |
| `lengkap` | Dokumen telah terverifikasi dan proposal siap ditugaskan untuk review. |

### Jenis review

Saat menetapkan proses review, Admin dapat memilih jenis review yang menentukan informasi yang dapat dilihat Reviewer.

| Jenis | Identitas pengusul terlihat | Dokumen dapat dibuka |
| --- | --- | --- |
| Open Review | Ya | Ya |
| Single Blind | Tidak | Ya |
| Double Blind | Tidak | Tidak |

Setiap perubahan status tercatat pada **Riwayat Pengajuan**, termasuk pelaku, status sebelum dan sesudah, pesan, serta waktu perubahan.
